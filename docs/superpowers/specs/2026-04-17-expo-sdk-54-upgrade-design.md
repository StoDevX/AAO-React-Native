# Expo SDK 53 → 54 Upgrade

**Status:** Design approved, awaiting implementation plan
**Date:** 2026-04-17
**Branch:** `claude/plan-expo-54-update-Ej26v`
**Prior art:** Commit `095f34a` (SDK 52→53, RN 0.76→0.79)

## Goal

Upgrade the app from Expo SDK 53 to SDK 54 (React Native 0.79.6 → 0.81.x,
React 19.0 → 19.1.x), staying on the Legacy Architecture, while folding in
the deprecation-driven `SafeAreaView` migration and the `react-native-screens`
floor required by the unstable native bottom-tabs navigator.

## Non-goals

- New Architecture opt-in (tracked separately in issue #7453; deferred to the
  SDK 55 cycle, which will require it).
- `react-native-reanimated` v3 → v4 (requires New Architecture).
- `react-native-screens` v5 evaluation.
- `expo-file-system` API migration (not a direct dep).
- Android changes of any kind (iOS-only app).
- Any unrelated refactors.

## Current state

- `expo@53.0.27`, `react-native@0.79.6`, `react@19.0.0`
- `react-native-screens@4.18.0` (bumped 2026-04-15 in PR #7505)
- `react-native-reanimated@3.19.5`
- iOS-only; `ENV['RCT_NEW_ARCH_ENABLED'] = '0'` in `ios/Podfile`
- Hermes is the default JS engine (no JSC usage)
- 26 internal packages in `modules/*` with `react` / `react-native` peer deps
- Node 22.22.2 via `.mise.toml` (≥20.19.4 ✓)

## Design

### Strategy: Legacy-Arch single-hop

Mirrors the SDK 52→53 upgrade pattern (commit `095f34a`). Keep the Podfile
New-Arch opt-out. Keep reanimated on v3. Do one clean SDK/RN/React bump plus
the deprecations it surfaces.

### Version bumps

Direct `package.json` dependencies:

| Package | From | To |
|---|---|---|
| `expo` | 53.0.27 | `^54.0.0` (resolve via `npx expo install --fix`) |
| `react` | 19.0.0 | 19.1.x |
| `react-native` | 0.79.6 | 0.81.x |
| `react-native-screens` | 4.18.0 | `^4.24.0` (latest 4.x at upgrade time) |
| `expo-asset` | 11.1.7 | SDK 54 pin |
| `expo-font` | 13.3.2 | SDK 54 pin |
| `expo-modules-core` | 2.5.0 | SDK 54 pin |
| `expo-web-browser` | 14.2.0 | SDK 54 pin |
| `@types/react` | 19.2.14 | React 19.1–matched version |
| `react-test-renderer` | 19.0.0 | 19.1.x |

Dev dependencies moving in lockstep with RN:

- `@react-native/babel-preset` → 0.81.x
- `@react-native/metro-config` → 0.81.x
- `@react-native/eslint-config` → 0.81.x (downgrade from 0.83.1 to match RN)

**Resolution mechanism:** trust `npx expo install --fix` for all `expo-*` and
bundled `react-native-*` packages. Take the exact versions it writes.

**Not bumped:** `react-native-reanimated` (stays on `3.19.5`); any package
outside Expo's dep map that `expo-doctor` does not flag.

### Code & native changes

#### React 19.1 / RN 0.81 fallout

- **Shims are expected to survive unchanged:** `types/jsx.d.ts` (global
  `JSX.*` re-export) and `contrib/0002-rn-abortsignal.patch` (drops RN's
  custom `AbortSignal`).
- If `contrib/0002-rn-abortsignal.patch` fails to apply, regenerate against
  RN 0.81's `AbortSignal.d.ts` using the same pattern.
- Fix whatever `tsc` surfaces. Expect small, localized hits.
- Fix whatever `react-test-renderer` 19.1 surfaces; `act()` adjustments only.

#### SafeAreaView migration (3 files)

Swap RN's `SafeAreaView` for the one from `react-native-safe-area-context`
(already a direct dep at `5.7.0`):

- `source/views/home/button.tsx` (lines 2, 19, 33)
- `source/views/directory/list.tsx` (lines 5, 143, 151)
- `source/views/settings/screens/api-test/list.tsx` (lines 2, 64, 74)

Drop-in import swap. The `withSafeAreaView` prop in `modules/tableview/index.tsx`
is unrelated to this migration — leave it alone.

#### iOS native project

- `ios/AllAboutOlaf/AppDelegate.swift`: no changes anticipated; add any new
  RN-0.81-required boilerplate if it exists.
- `ios/Podfile`: no changes. `RCT_NEW_ARCH_ENABLED = '0'` stays.
- `ios/Podfile.lock`: regenerated by `pod install`. Large churn expected.

#### Module peer deps (26 internal packages)

Scripted sweep across every `modules/*/package.json`:

- `react` peer → `^19.1.0`
- `react-native` peer → `^0.81.0`
- `expo-web-browser` peer in `modules/open-url` → SDK 54 version

#### `CLAUDE.md`

Bump the `React Native 0.79.6` version reference in the Project Overview
section to the new version. Required by the "Dependency upgrades" rule in
`CLAUDE.md` itself.

### Execution order

Each step leaves the repo committable. 💾 marks a checkpoint commit.

1. **Prep**
   - `mise run agent:setup`
   - Baseline: `mise run agent:pre-commit` passes on SDK 53 HEAD
2. **SDK bump**
   - `npx expo install expo@^54.0.0 --fix`
   - Manual edits for packages not in Expo's map (see table above)
   - `npx expo-doctor@latest`
   - Run `scripts/apply-patches.sh`; regenerate abortsignal patch if needed
3. **Module peer-dep sweep** across all `modules/*/package.json`
4. **💾 Commit A:** `upgrade React Native 0.79.6 → 0.81.x and Expo SDK 53 → 54`
5. **`mise run pods`**
   - Regenerate `ios/Podfile.lock`; add any new `AppDelegate.swift` lines RN
     0.81 requires
6. **💾 Commit B:** `chore: apply pod install changes for Expo SDK 54`
7. **Type & test fixes**
   - `mise run tsc` → fix inline
   - `mise run test` → fix inline
8. **SafeAreaView migration** (3 files)
   - Re-run `mise run tsc` and `mise run test`
9. **💾 Commit C:** `migrate SafeAreaView usages to react-native-safe-area-context`
10. **`CLAUDE.md` version bump** — folded into Commit A
11. **Final verification**
    - `mise run agent:pre-commit` green
    - `npx expo-doctor@latest` green
12. **Push & PR**

**Rollback:** each 💾 checkpoint is independently revertible.

### Acceptance criteria

The upgrade is done when all of the following hold:

1. **Version floor met**
   - `package.json`: `expo@^54.x`, `react-native@0.81.x`, `react@19.1.x`,
     `react-native-screens@>=4.24.0`
   - All 26 `modules/*/package.json` list `react@^19.1` and
     `react-native@^0.81` in peer deps where applicable
   - `CLAUDE.md` Project Overview shows RN `0.81.x`
2. **Static checks green**
   - `mise run agent:pre-commit` exits 0 (`pretty:check`, `lint`, `tsc`, `test`)
   - `npx expo-doctor@latest` has no errors. Warnings permitted only for
     explicitly-scoped-out packages (reanimated v3, screens major-version
     advisory), and recorded in the PR description.
3. **Native resolution verified**
   - `mise run pods` completes on Linux
   - `ios/Podfile.lock` regenerated and committed
   - `contrib/0002-rn-abortsignal.patch` applies cleanly
4. **SafeAreaView migration complete**
   - No imports of `SafeAreaView` from `'react-native'` in `source/` or
     `modules/` (the unrelated `withSafeAreaView` prop in
     `modules/tableview/index.tsx` is unaffected)
5. **Scope discipline**
   - `ios/Podfile` still contains `ENV['RCT_NEW_ARCH_ENABLED'] = '0'`
   - `react-native-reanimated` still at `3.19.5`
   - No files touched outside: `package.json`, `package-lock.json`,
     `modules/*/package.json`, `ios/AllAboutOlaf/AppDelegate.swift`,
     `ios/Podfile.lock`, `contrib/0002-rn-abortsignal.patch`, `CLAUDE.md`,
     the 3 SafeAreaView files, and whatever small `tsc`/test fixes the bump
     surfaces
6. **PR ready**
   - Branch pushed to `origin/claude/plan-expo-54-update-Ej26v`
   - PR body includes: SDK 54 changelog link, in-scope vs out-of-scope list,
     human-verification checklist (simulator build + smoke test of each of
     the 7 native-bottom-tab screens)
7. **Out-of-agent verification (blocks merge, not spec completion)**
   - A human runs the app on iOS simulator via Xcode 16.1+ and confirms it
     launches and the tab navigators render

## Risks & unknowns

### Known risks

- **`contrib/0002-rn-abortsignal.patch` may fail to apply.** RN 0.81 may have
  restructured the typings file. Mitigation: regenerate the patch against
  0.81's `AbortSignal.d.ts`.
- **React 19.1 `react-test-renderer` tightening.** SDK 53 upgrade already
  wrapped two snapshot tests in `act()`. Further adjustments possible.
- **`react-native-screens` 4.18 → 4.24+ in isolation.** Six minor versions
  in one jump. Native navigators are a classic "works in Metro, crashes on
  boot" source. The human-run iOS simulator build before merge is the
  safety net.
- **`expo-doctor` may flag non-Expo-map packages** (`react-native-webview`,
  `react-native-keychain`, etc.). Handle case-by-case; same-major nudges OK,
  major bumps are out of scope.
- **`@react-native/eslint-config` 0.83 → 0.81 downgrade.** Rules may shift.
  Fix violations inline; if the set is large, keep 0.83.1 (lint config's
  floor is React, not RN).

### Unknowns resolved during execution

- Exact resolved versions from `npx expo install --fix`
- Whether RN 0.81 requires new `AppDelegate.swift` lines
- Whether Metro 0.81 breaks `unstable_enablePackageExports` in
  `metro.config.js`
- Whether any `modules/*` imports a removed RN 0.81 API

### Stop-and-re-plan conditions

- `expo-doctor` reports an incompat that requires New Architecture to
  resolve (i.e., a pinned dep we can't hold at a Legacy-Arch-compatible
  version). Escalate; do not silently flip New Arch.
- RN 0.81 native changes extend beyond `AppDelegate.swift` +
  `Podfile.lock` (e.g., `.xcconfig`, `project.pbxproj` surgery). Require
  iOS simulator verification *before* Commit A, not at the end.
