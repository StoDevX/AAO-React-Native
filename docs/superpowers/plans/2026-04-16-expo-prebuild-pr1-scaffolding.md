# Expo Prebuild — PR 1 (Scaffolding) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land the Expo Prebuild scaffolding (`app.config.ts`, two library-provided config plugins, three local config plugins) on a dedicated branch, with unit tests for every local plugin, producing zero runtime behavior change. Existing `ios/` remains source-of-truth; `expo prebuild` is not invoked in this PR.

**Architecture:** Every iOS native customization in the current codebase is classified as (A) a declarative field on `app.config.ts`, (B) a library-provided config plugin, or (C) a local config plugin we author. This PR creates all three layers but does not invoke them — that happens in PR 2. Plugins are authored as pure helper functions wrapped by thin `@expo/config-plugins` adapters so each transformation can be unit-tested without standing up a full prebuild pipeline.

**Tech Stack:** TypeScript, Expo SDK 53, `@expo/config-plugins` 11.x (bundled with Expo 53), Jest + `@jest/globals`, `xcode` (npm — used internally by `@expo/config-plugins` for pbxproj mods, reused here in tests).

**Spec:** `docs/superpowers/specs/2026-04-16-expo-prebuild-migration-design.md` — sections "Config Plugin Inventory" (A/B/C/D) and "Migration Steps" (Step 0, Step 1, Step 2).

**Branch:** `claude/expo-prebuild-pr1-scaffolding` (created from `master` in Task 1). Not to be confused with `claude/expo-prebuild-migration-plan-2v3ST`, which holds the spec and this plan.

## File Structure

**New files (all live on the implementation branch):**

| Path | Responsibility |
|---|---|
| `app.config.ts` | Declares app metadata + `ios.infoPlist` overrides + `plugins: [...]` array. Single source of truth for iOS native config. |
| `plugins/types.ts` | Shared type aliases for plugin authors (`ConfigPlugin`, config shapes, etc.). |
| `plugins/with-app-delegate-customizations.ts` | Inserts `URLCache` sizing, `AVAudioSession.playback`, and `--reset-state` handler into `AppDelegate.swift` via `withAppDelegate`. Exports a pure helper `patchAppDelegate(source: string): string` for testing. |
| `plugins/__tests__/with-app-delegate-customizations.test.ts` | Unit tests for `patchAppDelegate`. |
| `plugins/with-alternate-icons.ts` | Adds `CFBundleIcons.CFBundleAlternateIcons.icon_type_windmill` to `Info.plist` (both iPhone and iPad), and references the four `windmill*.png` files as iOS bundle resources. Pure helpers: `addAlternateIcons(plist: object): object` and `addWindmillResources(project: xcode.XcodeProject): xcode.XcodeProject`. |
| `plugins/__tests__/with-alternate-icons.test.ts` | Unit tests for both helpers. |
| `plugins/with-xcuitest-target.ts` | Reinstates the `AllAboutOlafUITests` target in `project.pbxproj`, adds `inherit! :none` to its Podfile target, and injects the `ExpoUITestsAutolinkingFix` monkey-patch at the top of the Podfile. Pure helpers: `ensureUITestTarget(project, opts): project`, `patchPodfileForUITests(source: string): string`. |
| `plugins/__tests__/with-xcuitest-target.test.ts` | Unit tests for both helpers. |
| `__tests__/app-config.test.ts` | Imports `app.config.ts` and asserts on its exported shape (declarative fields + plugins list). |
| `docs/superpowers/plans/2026-04-16-expo-prebuild-pr1-scaffolding.md` | This plan. Already committed to the planning branch before implementation starts. |

### 0.1 Check `react-native-change-icon` for an Expo config plugin

- [ ] **Step 1: Check the installed version's package for a plugin**

Run:
```bash
cat node_modules/react-native-change-icon/package.json | grep -E '"expo"|"main"|"version"'
ls node_modules/react-native-change-icon/ | grep -iE 'plugin|app\.plugin'
```

Expected: either `app.plugin.js` / `expo.plugin` / `plugin/` directory is present (library ships a plugin), or nothing matches (library does not ship a plugin).

- [ ] **Step 2: If not bundled, check latest release on npm**

Run:
```bash
npm view react-native-change-icon@latest files
npm view react-native-change-icon@latest version
```

Expected: the `files` list does or does not include `app.plugin.js` or a `plugin/` directory.

- [ ] **Step 3: Record finding**

Create a scratch note (not committed) with one of:
- **Finding A:** library ships a config plugin in installed version → **Task 5 is replaced** by "add the library's plugin to `app.config.ts`".
- **Finding B:** library ships a plugin in a newer version we can upgrade to → **Task 5 is replaced**, and plan a version bump.
- **Finding C:** no plugin available → **Task 5 proceeds as written** (author `with-alternate-icons.ts` locally).
